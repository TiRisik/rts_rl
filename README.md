# RTS-среда для обучения с подкреплением

Пошаговая стратегия (Real-Time Strategy) в виде gym-совместимой среды для сравнения алгоритмов глубокого обучения с подкреплением: **PPO**, **A2C** и **DQN**. Проект предназначен для исследования поведения RL-агентов в задачах управления ресурсами, производством юнитов и тактическим боем.

---

## Структура проекта

| Файл | Назначение |
|---|---|
| `consts.py` | Константы: размеры сетки, цвета, параметры экрана |
| `units.py` | Классы игровых юнитов: `Worker`, `Light`, `Heavy`, `Ranged` |
| `buildings.py` | Классы зданий: `Base`, `Barracks`, `Factory`, `Tower` |
| `recources.py` | Классы ресурсов: `Minerals`, `Gas` |
| `main.py` | Основной класс игры `RTSGame`  |
| `rts_env.py` | Gymnasium-среда `RTSEnv` — обёртка над игрой для RL-обучения |
| `train_ppo.py` | Обучение агента методом PPO (Proximal Policy Optimization) |
| `train_a2c.py` | Обучение агента методом A2C (Advantage Actor-Critic) |
| `train_dqn.py` | Обучение агента методом DQN (Deep Q-Network) |
| `evaluate_all.py` | Оценка обученных моделей и построение сравнительных графиков |

## Игровая механика

Поле представляет собой сетку **25 × 20** клеток. Два игрока — **синий** (управляется агентом) и **красный** (встроенный противник) — развивают экономику и ведут боевые действия в пошаговом режиме. Цель — уничтожить базу противника.

### Типы юнитов

| Юнит | HP | Атака | Дальность атаки | Скорость | Стоимость |
|---|---|---|---|---|---|
| Рабочий (`Worker`) | 30 | 5 | 1 | 2 | 25 минералов |
| Лёгкий (`Light`) | 50 | 15 | 2 | 3 | 50 минералов |
| Тяжёлый (`Heavy`) | 120 | 30 | 1 | 1 | 100 минералов + 25 газа |
| Дальний (`Ranged`) | 40 | 20 | 4 | 2 | 75 минералов + 50 газа |

### Типы зданий

| Здание | HP | Функция |
|---|---|---|
| База (`Base`) | 500 | Производство рабочих; потеря = поражение |
| Казарма (`Barracks`) | 200 | Производство боевых юнитов |
| Башня (`Tower`) | 150 | Автоматически атакует врагов в радиусе 2 клетки |
| Фабрика (`Factory`) | 250 | Расширенное производство (зарезервировано) |

### Ресурсы

- **Минералы** (`Minerals`) — основной ресурс для строительства и производства юнитов.
- **Газ** (`Gas`) — редкий ресурс, необходимый для производства тяжёлых и дальних юнитов.

Рабочие автоматически собирают ресурсы при назначении цели добычи.

## Среда (`RTSEnv`)

### Пространство наблюдений



Наблюдение представляет собой **плоский вектор** размером `25 × 20 × 22 = 11 000` элементов, получаемый из трёхмерного тензора формы `(высота, ширина, признаки)`. Каждая клетка поля описывается **22 признаками**:

| Индекс | Признак | Описание |
|---|---|---|
| 0 | `unit_hp` | Очки здоровья юнита (нормировано на `max_hp`) |
| 1 | `building_hp` | Очки здоровья здания (нормировано на `max_hp`) |
| 2 | `blue_unit` | Флаг: в клетке стоит юнит синего |
| 3 | `red_unit` | Флаг: в клетке стоит юнит красного |
| 4 | `blue_base` | Флаг: в клетке стоит база синего |
| 5 | `red_base` | Флаг: в клетке стоит база красного |
| 6 | `blue_barracks` | Флаг: казарма синего |
| 7 | `red_barracks` | Флаг: казарма красного |
| 8 | `blue_tower` | Флаг: башня синего |
| 9 | `red_tower` | Флаг: башня красного |
| 10 | `minerals` | Количество минералов в клетке (нормировано на 400) |
| 11 | `gas` | Количество газа в клетке (нормировано на 400) |
| 12 | `is_worker` | Флаг: юнит является рабочим |
| 13 | `is_light` | Флаг: юнит является лёгким |
| 14 | `is_heavy` | Флаг: юнит является тяжёлым |
| 15 | `is_ranged` | Флаг: юнит является дальним |
| 16 | `has_acted` | Флаг: юнит уже совершил действие в этом ходу |
| 17 | `blue_minerals` | Минералы синего (глобально, нормировано на 500) |
| 18 | `blue_gas` | Газ синего (глобально, нормировано на 200) |
| 19 | `pending_build` | Прогресс незавершённого строительства (нормировано на 3 хода) |
| 20 | `barracks_busy` | Флаг: казарма синего занята производством |
| 21 | `red_minerals` | Минералы красного (глобально, нормировано на 500) |

Все значения нормированы в диапазон `[0, 1]`.

### Пространство действий


Агент выбирает одно из **14 дискретных макро-действий**, применяемых ко всем доступным юнитам синего одновременно:

| ID | Действие | Описание |
|---|---|---|
| 0 | Постройка башни | Рабочий строит башню вблизи своей позиции (100 минералов + 50 газа) |
| 1 | Постройка казармы | Рабочий строит казарму (150 минералов, требует наличие базы) |
| 2 | Создать рабочего | База производит рабочего (25 минералов) |
| 3 | Создать дальнего | Казарма производит дальнего юнита (75 минералов + 50 газа) |
| 4 | Создать лёгкого | Казарма производит лёгкого юнита (50 минералов) |
| 5 | Создать тяжёлого | Казарма производит тяжёлого юнита (100 минералов + 25 газа) |
| 6 | Бойцы → база врага | Все бойцы движутся к базе противника |
| 7 | Бойцы → казарма врага | Все бойцы движутся к казарме противника |
| 8 | Бойцы → ближайший враг | Каждый боец движется к ближайшему вражескому юниту |
| 9 | Атака врага в радиусе | Бойцы атакуют врагов в пределах своей дальности атаки |
| 10 | Атака базы врага | Бойцы атакуют базу противника |
| 11 | Атака казарм врага | Бойцы атакуют казармы противника |
| 12 | Рабочие → минералы | Рабочие направляются к ближайшему месторождению минералов |
| 13 | Рабочие → газ | Рабочие направляются к ближайшему источнику газа |

### Функция вознаграждения

Вознаграждение формируется каждый шаг как взвешенная сумма следующих компонентов:

| Компонент | Значение | Описание |
|---|---|---|
| Шаговый штраф | −0.01 | Небольшой штраф за каждый ход (стимулирует скорость) |
| Уничтожение вражеского юнита | +1.5 за юнита | Награда за каждого убитого противника |
| Потеря своего юнита | −0.3 за юнита | Штраф за потерю юнитов синего |
| Урон по базе врага | +0.01 × урон | Поощрение за нанесение урона базе |
| Суммарный урон врагу | +0.005 × урон | Поощрение за любой нанесённый урон |
| Уничтожение здания врага | +3.0 за здание | Крупная награда за разрушение вражеских построек |
| Потеря своего здания | −1.0 за здание | Штраф за потерю своих зданий |
| Новый боец | +0.4 за юнита | Награда за производство боевого юнита |
| Сбор минералов | +0.005 × кол-во × масштаб | Награда за добычу (масштаб снижается по мере роста армии) |
| Сбор газа | +0.01 × кол-во × масштаб | Награда за добычу газа |
| Трата ресурсов | +0.01–0.015 × трата | Поощрение за активное использование ресурсов |
| Сближение с базой врага | +0.06 × улучшение | Награда за движение бойцов в сторону базы противника |
| Невалидное действие | −0.05 | Штраф за действие без нужных ресурсов или юнитов |
| **Победа** | **+20.0** | Терминальная награда за уничтожение базы врага |
| **Поражение** | **−10.0** | Терминальный штраф за потерю своей базы |
| **Ничья** (превышен лимит) | **−5.0** | Штраф за истечение лимита в 100 ходов |

## Алгоритмы обучения

Все алгоритмы реализованы через библиотеку [Stable-Baselines3](https://stable-baselines3.readthedocs.io/) и обучаются на **500 000 шагов** с одинаковой архитектурой нейросети.

### Общие параметры

- Количество шагов обучения: `500 000`
- Частота оценки: каждые `50 000` шагов (5 эпизодов)
- Сохранение чекпоинтов: каждые `100 000` шагов
- Коэффициент дисконтирования (γ): `0.995`

### PPO (Proximal Policy Optimization)

```
Параллельные среды : 4
n_steps            : 1024
batch_size         : 128
n_epochs           : 4
learning_rate      : 3e-4
clip_range         : 0.2
ent_coef           : 0.03
gae_lambda         : 0.95
Архитектура        : pi=[256, 256, 128], vf=[256, 256, 128]
```

### A2C (Advantage Actor-Critic)

```
Параллельные среды : 4
n_steps            : 5
learning_rate      : 7e-4
gae_lambda         : 1.0
ent_coef           : 0.01
Архитектура        : pi=[256, 256, 128], vf=[256, 256, 128]
```

### DQN (Deep Q-Network)

```
Параллельные среды : 1 (DQN не поддерживает параллелизм)
buffer_size        : 10 000
learning_starts    : 5 000
learning_rate      : 1e-4
exploration_fraction: 0.2 (eps: 1.0 → 0.05)
target_update_interval: 1 000
Архитектура        : [256, 256, 128]
```

## Оценка моделей (`evaluate_all.py`)

Скрипт загружает обученные модели и прогоняет каждую на `1000` эпизодах, собирая следующую статистику:

- **Процент побед / поражений / ничьих**
- **Суммарная и средняя награда** (со скользящим средним MA-10)
- **Средняя длина партии** (в ходах)
- **Распределение выбранных действий**
- **Динамика количества юнитов и ресурсов** по ходам

Результаты сохраняются в виде:
- `evaluation_report_ppo.png` / `_a2c.png` / `_dqn.png` — индивидуальные отчёты по каждой модели
- `evaluation_comparison.png` — сравнительный отчёт по всем алгоритмам

## Запуск

### 1. Обучение модели PPO

```bash
python train_ppo.py
```

Сохраняет модель в `rts_ppo_model.zip`, логи — в `./logs/`.

### 2. Обучение модели A2C

```bash
python train_a2c.py
```

Сохраняет модель в `rts_a2c_model.zip`, логи — в `./logs_a2c/`.

### 3. Обучение модели DQN

```bash
python train_dqn.py
```

Сохраняет модель в `rts_dqn_model.zip`, логи — в `./logs_dqn/`.

### 4. Оценка и сравнение моделей

```bash
python evaluate_all.py
```

Аргументы командной строки:

| Аргумент | Описание | По умолчанию |
|---|---|---|
| `--n N` | Количество партий на модель | 1000 |
| `--ppo PATH` | Путь к PPO-модели (без .zip) | `rts_ppo_model` |
| `--a2c PATH` | Путь к A2C-модели (без .zip) | `rts_a2c_model` |
| `--dqn PATH` | Путь к DQN-модели (без .zip) | `rts_dqn_model` |
| `--best` | Использовать `best_model.zip` из папки логов | выключено |
| `--out FILE` | Имя PNG сравнительного отчёта | `evaluation_comparison.png` |
| `--txt FILE` | Имя текстового отчёта | `evaluation_report.txt` |

Пример:
```bash
python evaluate_all.py --n 200 --best
```
